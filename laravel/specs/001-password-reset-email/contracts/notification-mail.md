# Notification & Mail Contract — Password Reset by Email

A single notification class delivers the reset email. The contract below pins what the mail must contain, the delivery channel, and how it is enqueued.

## Class

`App\Notifications\PasswordResetRequested implements ShouldQueue`

```php
namespace App\Notifications;

use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Notifications\Messages\MailMessage;
use Illuminate\Notifications\Notification;
use Illuminate\Support\Facades\URL;

class PasswordResetRequested extends Notification implements ShouldQueue
{
    use Queueable;

    public function __construct(public string $token) {}

    public function via(object $notifiable): array
    {
        return ['mail'];
    }

    public function toMail(object $notifiable): MailMessage
    {
        $url = URL::temporarySignedRoute(
            'password.reset',
            now()->addMinutes(60),
            ['token' => $this->token, 'email' => $notifiable->email],
        );

        return (new MailMessage)
            ->subject(__('auth.mail.reset.subject'))
            ->markdown('mail.auth.password-reset', [
                'url' => $url,
                'expiresInMinutes' => 60,
                'recipientName' => $notifiable->name,
            ]);
    }
}
```

## Channels

- `mail` only. (Future channels: `nexmo`/SMS or `database`/in-app notification — not in scope.)

## Queue

- Default connection (`QUEUE_CONNECTION=database` per `.env.example`).
- Queue name `mail` (Laravel default for `MailMessage::onQueue(...)` is unset, defaulting to `default`; we explicitly leave it on `default` to avoid creating a queue worker requirement; the existing `make up-workers` target already runs `queue:listen` against `default`).
- `tries = 3`, `backoff = [10, 60, 300]` (declared on the class).

## Mail content contract

The Blade view at `resources/views/mail/auth/password-reset.blade.php` MUST:

1. **Open with a greeting** that uses `$recipientName`. PT-BR: "Olá, {{ $recipientName }}." Fallback to "Olá." if name is blank.
2. **State the reason** in one sentence: "Recebemos um pedido de redefinição de senha para sua conta."
3. **Embed the action button** linking to `$url` with the label "Criar nova senha" (PT-BR) / "Set new password" (EN).
4. **State the expiration** explicitly: "Este link expira em {{ $expiresInMinutes }} minutos." (PT-BR) — REQUIRED by FR-005.
5. **State that the email can be ignored** if not requested: "Se você não solicitou esta redefinição, pode ignorar este e-mail. Sua senha atual continua válida." — REQUIRED by FR-005.
6. **Sign off** with the product brand line (the `App\Providers\AppServiceProvider` or `config/app.name` value).
7. **Plain-text fallback** auto-generated by Laravel's markdown mailer; no extra view needed.

## Subject contract

Single line, PT-BR by default:

> "Redefinição de senha — Geffin"

EN fallback:

> "Reset your Geffin password"

Stored in `lang/{locale}/auth.php` under `mail.reset.subject`.

## Sender contract

- From: `config('mail.from.address')` / `config('mail.from.name')` — `.env.example` already declares `noreply@geffin.local` / `${APP_NAME}`. No per-notification override.
- Reply-To: not set (the existing default applies).

## URL contract

- Route: `password.reset` (name).
- Params: `token` (path) + `email` (query).
- Signed: yes — `URL::temporarySignedRoute(..., now()->addMinutes(60), ...)`. Expiry matches token expiry so a tampered URL is rejected by signature middleware **before** the broker is consulted; an expired but untampered URL is rejected by signature middleware OR by the broker, whichever fires first. Both cases satisfy the spec's "tampered → invalid link error" requirement (FR-015 edge case).

## Testing contract

In `Notification::fake()` mode the assertions test infrastructure must make:

- `Notification::assertSentTo($user, PasswordResetRequested::class)` — sent.
- `Notification::assertNothingSent()` on the unknown-email branch.
- Inspect the notification's `toMail($user)` output — assert subject string, presence of `$url` in the rendered Blade, presence of the expiration sentence, presence of the "ignore this email" sentence.

## Localization

- Notification's locale defaults to `pt_BR` when `$notifiable->preferred_locale` is unset (the column does not exist yet; the notification reads via `$notifiable->preferred_locale ?? 'pt_BR'`).
- Laravel localizes `__()` calls inside `toMail` automatically because `Notification` propagates the locale into the queued job's payload.

## Out of scope

- Bounce / soft-fail handling: the spec calls out that bounces should be logged but not surfaced. This requires a webhook from the eventual mail provider (e.g. SES, Postmark) which is not configured today. A `// TODO(bounce)` is left in the notification class for the future hook that writes an `email_sent:failed` audit row.
